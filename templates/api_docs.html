🚀 API Documentation

Access TheraSwitchRx's powerful medicine recommendation engine through our REST API. Get AI-powered alternatives and detailed medicine information programmatically.

⚡ Quick Start

Get started with the TheraSwitchRx API in just 3 steps:

  1. Get your API key: Register here
  2. Make your first request
  3. Build amazing healthcare apps!
// TheraSwitchRx API - JavaScript Example
const response = await fetch('http://localhost:5000/api/v1/search', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'X-API-Key': 'tsx_your_api_key_here'
    },
    body: JSON.stringify({
        query: 'alternatives for Paracetamol'
    })
});

const data = await response.json();
console.log(data.data.recommendation);
# TheraSwitchRx API - Python Example
import requests

response = requests.post(
    'http://localhost:5000/api/v1/search',
    headers={'X-API-Key': 'tsx_your_api_key_here'},
    json={'query': 'alternatives for Paracetamol'}
)

data = response.json()
print(data['data']['recommendation'])
# TheraSwitchRx API - cURL Example
curl -X POST http://localhost:5000/api/v1/search \
  -H "Content-Type: application/json" \
  -H "X-API-Key: tsx_your_api_key_here" \
  -d '{"query": "alternatives for Paracetamol"}'

🔗 Base URL

All API requests should be made to:

http://localhost:5000/api/v1

🔐 Authentication

Most endpoints require an API key. Include your API key in the request headers:

X-API-Key: tsx_your_api_key_here

Or as an Authorization header:

Authorization: Bearer tsx_your_api_key_here

📋 API Endpoints

POST /search 🔐 Auth Required

Description: Get AI-powered medicine alternatives and recommendations

Request Body:

{
  "query": "alternatives for Paracetamol"
}

Response:

{
  "success": true,
  "data": {
    "query": "alternatives for Paracetamol",
    "recommendation": "AI-generated recommendation text...",
    "metadata": {
      "timestamp": "2024-01-01T12:00:00",
      "request_id": "uuid-here",
      "database_size": "6+ Lakh medicines",
      "processing_type": "AI-powered analysis"
    }
  },
  "message": "Recommendations generated successfully",
  "api_usage": {
    "requests_remaining": 95,
    "plan": "free"
  }
}

GET /medicine/{medicine_name} 🔐 Auth Required

Description: Get detailed information about a specific medicine

Example: /api/v1/medicine/Paracetamol

Response:

{
  "success": true,
  "data": {
    "medicine_name": "Paracetamol",
    "information": "Detailed medicine information...",
    "timestamp": "2024-01-01T12:00:00"
  },
  "message": "Information retrieved successfully"
}

POST /alternatives 🔐 Auth Required

Description: Get alternatives for multiple medicines (max 10 per request)

Request Body:

{
  "medicines": ["Paracetamol", "Crocin", "Ibuprofen"]
}

Response:

{
  "success": true,
  "data": {
    "results": {
      "Paracetamol": {
        "alternatives": "Alternative medicines...",
        "status": "success"
      }
    },
    "processed_count": 3
  }
}

⚠️ Error Handling

All errors return a consistent format:

{
  "success": false,
  "error": "Error message",
  "code": "ERROR_CODE"
}

Common Error Codes:

  • MISSING_API_KEY - API key not provided
  • INVALID_API_KEY - API key is invalid or expired
  • RATE_LIMIT_EXCEEDED - Daily limit exceeded
  • MISSING_QUERY - Required query parameter missing
  • INTERNAL_ERROR - Server error occurred

💡 Best Practices

  • Cache responses when possible to reduce API calls
  • Handle errors gracefully with proper error checking
  • Use specific queries for better AI recommendations
  • Monitor your usage with the /key-info endpoint
  • Set timeouts for your HTTP requests (recommend 30 seconds)
  • Use HTTPS in production environments
